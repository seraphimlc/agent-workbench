# Runtime Foundation Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first independently runnable part of Foundation Slice 1: an authenticated local Daemon with a versioned protocol, SQLite-backed Workspace/Session/Turn/Event persistence, and one durable scheduler slot.

**Architecture:** A strict shared protocol package defines length-prefixed RPC envelopes. A Node Daemon owns the Unix socket and better-sqlite3 connection. This subplan stops before Model Gateway, Runner execution, file tools, Artifact, Keychain integration, and Electron; those attach to the stable contracts created here in subsequent plans. The development smoke client supplies the bootstrap secret through file descriptor 3 so it never appears in argv or environment; Electron Main will replace that source with Keychain in the desktop plan.

**Tech Stack:** TypeScript 5.9, pnpm 10, Zod 4, better-sqlite3 12, Vitest 4, Node Unix domain sockets, HMAC-SHA256.

---

## Scope and test list

This plan must prove:

- the empty repository installs deterministically and can run test/typecheck/lint/build commands;
- protocol envelopes cover request, response, notification, cancel, heartbeat, errors, Session Events, and Slice 1A methods;
- framing is 4-byte big-endian length + UTF-8 JSON, with 16 MiB and depth-64 limits;
- an unauthenticated connection cannot call `app.health`;
- challenge-response uses a single-use nonce and a 256-bit secret supplied outside argv/env;
- Workspace, Session, first user Message, Turn, and Events persist across Daemon restart;
- Session Event seq is gap-free and `event.listAfter` returns a continuous projection;
- one persistent scheduler slot allows one running Turn, keeps later Turns queued, and is recovered honestly after restart;
- duplicate mutation `clientRequestId` returns the original result rather than creating duplicate facts.

Not in this plan: Keychain, Model Profile, Session Runner, ToolRun, Audit intent, files, Artifact, Electron, live subscribe, cancellation, or recovery UI.

  ```ts
  type RequestEnvelope = {
    kind: "request";
    protocolVersion: 1;
    requestId: string;
    traceId: string;
    sessionId: string | null;
    turnId: string | null;
    method: RpcMethod;
    payload: unknown;
    clientRequestId: string | null;
  };

  type ResponseEnvelope =
    | { kind: "response"; protocolVersion: 1; requestId: string; traceId: string; ok: true; result: unknown }
    | { kind: "response"; protocolVersion: 1; requestId: string; traceId: string; ok: false; error: ErrorEnvelope };

  type NotificationEnvelope = {
    kind: "notification";
    protocolVersion: 1;
    traceId: string;
    method: "auth.challenge" | "event" | "resync_required";
    payload: unknown;
  };

  type CancelEnvelope = { kind: "cancel"; protocolVersion: 1; traceId: string; targetRequestId: string };
  type HeartbeatEnvelope = { kind: "heartbeat"; protocolVersion: 1; traceId: string; daemonEpoch: string; leaseEpoch: number };
  ```

  Slice 1A methods and payload schemas:

  - `auth.respond { nonce, mac }` (first request only);
  - `app.health {}`;
  - `workspace.register { path }`;
  - `session.create { workspaceId, title, prompt }`;

  - `turn.enqueue { sessionId, prompt }`;

  `SessionSnapshot` fields for this plan: Session row, ordered Messages, ordered Turns, highWaterSeq, and Events through highWaterSeq.

- [ ] **Step 3: Implement the codec**

It must never use newline framing.
